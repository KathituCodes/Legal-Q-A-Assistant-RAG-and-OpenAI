# Legal Q&A Assistant (RAG + OpenAI)

A question-answering tool for legal documents. Ask plain-English questions about contracts and get precise, context-aware answers — without reading hundreds of pages yourself.

Built with **Retrieval-Augmented Generation (RAG)**: the system searches a database of legal text for the most relevant passages, then uses GPT to turn those passages into a clear answer.

---

## What it does

You type a question like:

> *"What are the termination conditions in this contract?"*

The assistant searches 510 real legal contracts from the CUAD dataset, finds the most relevant clauses, and returns a direct answer with the source passages it used.

---

## How it works (plain English)

The pipeline has four stages:

**1. Load and chunk**
The CUAD dataset (510 annotated legal contracts) is loaded and split into overlapping 1,000-token passages. Overlap ensures that clauses spanning a chunk boundary are not cut off mid-sentence.

**2. Embed and store**
Each passage is converted into a numerical vector using OpenAI's embedding model. All vectors are stored in a FAISS index — a fast similarity search database that lives on disk so you only build it once.

**3. Retrieve**
When you ask a question, your question is also converted into a vector. FAISS finds the 5 passages whose vectors are closest in meaning to your question.

**4. Generate**
Those 5 passages plus your question are sent to GPT-3.5-turbo. GPT reads only those passages and writes a focused answer grounded in the actual contract text.

## Requirements

- A Google Colab account (free tier works)
- An OpenAI API key with access to `gpt-3.5-turbo` and the embeddings API
- A stable internet connection for the first run (dataset download + embedding build)

> **Cost note:** Building the FAISS index embeds approximately 50,000 text chunks via the OpenAI API. This takes 10 to 30 minutes and costs approximately $1 to $3 on first run. After that, the index is saved to disk and subsequent runs are instant and free.

## Setup

### Step 1 — Get your OpenAI API key

1. Go to [platform.openai.com](https://platform.openai.com)
2. Click your profile icon, then **API keys**
3. Create a new key and copy it

### Step 2 — Add the key to Colab Secrets

This keeps your key out of the notebook code.

1. Open the notebook in Google Colab
2. Click the **key icon** in the left sidebar (Secrets)
3. Click **Add new secret**
4. Set the name to exactly: `OPENAI_API_KEY`
5. Paste your key as the value
6. Toggle **Notebook access** on

### Step 3 — Run the notebook

Run all cells from top to bottom. Each cell has a comment explaining what it does.

On first run, the FAISS index build cell will print progress and tell you when it is done. On every run after that, it loads from disk in seconds.

---

## Using the Streamlit app

The second-to-last cell in the notebook writes `app.py` and the final cell launches it with a public URL via pyngrok.

1. Run all notebook cells in order
2. The last cell prints a line like:

   ```
   Legal Q&A Assistant is live at: NgrokTunnel: "https://xxxx.ngrok.io"
   ```

3. Click that URL to open the app in your browser
4. Type your question in the text box and press Enter
5. The answer appears below, with an expandable section showing the exact contract passages the answer was drawn from

The app stays live as long as the Colab runtime is running.

## Troubleshooting

**"OPENAI_API_KEY environment variable is not set"**
You skipped Step 2, or toggled Notebook access off. Go back to Colab Secrets, confirm the key is added, and make sure Notebook access is enabled, then re-run from the API key cell.

**"KeyError: 'text'"**
The dataset column names have changed in a newer version of the parquet file. Run `print(df.columns.tolist())` in the load cell to see the actual names, then update `df['text']` to match.

**The FAISS build cell has been running for over 30 minutes**
This is expected on the full dataset. To test faster, add `chunks = chunks[:500]` directly before the `FAISS.from_documents(...)` line to build on a subset first. Delete the `faiss_legal_index/` folder when you are ready to build the full index.

**The pyngrok URL is not opening**
The Colab runtime has likely disconnected or timed out. Re-run all cells from the top. The FAISS index is already saved so the expensive build step will be skipped automatically.

**Answers seem vague or unrelated to your question**
Rephrase using specific legal terms. For example, use "indemnification clause" rather than "who pays if something goes wrong." More specific queries return tighter, more relevant passages from the vector search.
